# RobotScheduler - 机器人任务调度与路径规划系统

## 项目简介

RobotScheduler 是一个**大规模机器人任务调度与路径规划系统**，能够在10000×10000的网格地图上同时管理1000-3000个机器人，处理60-15000个任务，并智能避开5000-20000个障碍物。

### 主要特性

- 🤖 **大规模并发**：支持数千个机器人同时协同工作
- 🎯 **智能任务分配**：自动将任务分配给最近的空闲机器人
- 🗺️ **A*路径规划**：基于A*算法的高效避障路径规划
- ⚡ **多线程架构**：任务生成、调度、执行完全并行
- 📊 **实时可视化**：直观展示机器人状态和任务执行进度
- 🔧 **高性能优化**：采用地图分块管理，优化大规模场景性能

## 系统架构

```
Frontend (Qt) ──→ Backend (Coordinator, PathPlanner, Scheduler) ──→ Core (Map, Robot, Task)
```

- **Frontend**: 基于Qt的图形用户界面，包括参数配置和实时监控
- **Backend**: 核心调度逻辑，包括协调器、路径规划器和调度器
- **Core**: 基础数据结构和地图管理

## 技术栈

- **语言**: C++17
- **UI框架**: Qt5
- **算法**: A*路径规划、贪心任务分配
- **并发**: C++标准线程库（thread、mutex、atomic）

## 快速开始

### 构建项目

```bash
mkdir build
cd build
cmake ..
make
```

### 运行程序

```bash
./RobotScheduler
```

### 使用说明

1. 在启动界面输入参数：
   - 机器人数量：1000-3000
   - 障碍物数量：5000-20000
   - 任务数量：60-15000

2. 点击"开始任务"进入监控界面

3. 实时观察机器人执行任务的过程

## 核心算法

### A*路径规划
- 使用曼哈顿距离作为启发式函数
- 优先队列优化节点选择
- 四方向移动（上、下、左、右）
- 自动避开障碍物

### 任务调度
- 最近空闲机器人优先策略
- 多线程并发任务生成
- 实时任务分配和执行监控

### 地图管理
- 分块策略优化大规模地图
- 8×8格子为一个块（1250×1250块）
- 快速位置查询和碰撞检测
